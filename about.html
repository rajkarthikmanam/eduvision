Logical Data Model (Look & Feel)

The design follows two one-to-many relationships: Student → Enrollment and Course → Enrollment. Each student can enroll in multiple courses, and each course can have many students.

erDiagram
  STUDENT ||--o{ ENROLLMENT : enrolls
  COURSE  ||--o{ ENROLLMENT : has

  STUDENT {
    int id PK
    string name
    string major
    int age
    decimal gpa
  }

  COURSE {
    int id PK
    string code
    string title
    int credits
    string dept
  }

  ENROLLMENT {
    int id PK
    int studentId FK
    int courseId  FK
    string term
    decimal numeric_grade
  }
